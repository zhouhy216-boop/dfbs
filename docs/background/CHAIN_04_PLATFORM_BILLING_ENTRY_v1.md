## Main Object
- object_name: 平台来源报价单
- source: 平台费计费提醒结果 / 平台后付费上传记录
- result_on_completed: 形成平台来源报价单，进入后续报价单体系

## Trigger
- 本链讨论预付费、后付费、免费三类平台口径
- 平台费和SIM卡费分开生成来源报价单
- 免费平台不生成收费来源报价单
- 后续是否与其他来源报价单合并，由报价单体系决定

## Roles
- 平台管理员
  - actions: 上传点数, 核实平台侧情况, 确认收费建议, 发起平台来源报价单
  
## Free Rule
- 免费平台分为两类：
  - 公司内部使用免费
  - 销售申请免费
- 公司内部使用免费不走申请流程
- 由管理员直接设置免费，并填写免费原因
- 销售申请免费不在系统内发起
- 以线下签完的单据作为依据
- 系统只承接上传后的免费依据留存
- 由管理员根据线下依据设置免费
- 免费平台不生成收费来源报价单
- 免费平台不进入对客收费报价链
- 当前不展开线下单据模板、审批流、页面入口与状态流  

## Billing Mode Rule
- 平台收费分为：预付费 / 后付费 / 免费
- 预付费、后付费属于收费口径，进入平台来源报价单体系
- 免费属于平台存在但不收费的口径
- 免费平台由本链承接记录与追溯边界，但不生成收费来源报价单

## Start Rule
- 平台费限免期的起算节点，与合同约定的保修起算节点相同
- 保修起算节点当前可为：发货 / 发船 / 签收 / 调试完成
- 平台费限免期在合同中允许原样填写
- 系统统一将平台费限免期折算为月
- 到达限免期结束后，系统进入平台收费提醒与建议流程

## Prepaid Rule
- 系统不精确自动算费
- 系统只做提醒与建议
- 原因：
  - 系统内看不到销量
  - 平台管理员上传的是“上个月每个平台、每个机构的总点数”
  - 上传点数可能包含仍在限免期机器产生的点数
  - 上传点数中没有机器号

## Prepaid Reminder Rule
- 预付费下至少两类提醒：
  - 下个月有设备到达限免期，提醒平台主管准备收费
  - 上个月产生点数已大于剩余点数，提醒平台主管准备收费

## Prepaid Suggestion Rule
- 系统参考去年该机构全年实际点数，给出下一年预收建议
- 建议规则：
  - 去年总点数 / 12
  - 月均点数向上取整
  - 再按12个月预收下一年
- 示例：
  - 去年全年22点
  - 22 / 12 = 1.83
  - 向上取整为2
  - 下一年建议预收24点
- 系统给出的是建议，不是硬收费结果
- 最终收费方案由平台主管与客户确认
- 客户可不按建议缴费，也允许欠点

## Year-End Rule
- 一般不管点数是否用完，年底都会发起一次收费
- 系统应支持年底集中收费的业务倾向
- 但最终仍以人工与客户确认结果为准

## Postpaid Rule
- 后付费平台金额来自固定格式上传记录
- 上传记录中包含金额
- 系统根据上传记录整理并生成平台来源报价单
- 后付费也进入报价单体系，以统一归口和入账

## Quote Item Rule
- 一个客户下多个机构时
- 平台费用按机构拆成多条报价项
- 再进入后续报价单体系

## Quote Content Rule
- 平台来源报价项中应说明：
  - 去年购买多少点
  - 实际使用多少点
  - 剩余多少点 / 欠额多少点
  - 本次建议补多少点

## Derived Rules
- 平台计点规则在智能化平台内，系统内不可见
- 系统看到的是上传后的机构总点数，不是销量明细
- 系统提醒后，平台主管仍需到智能化平台侧核实并调整
- 平台来源报价单先独立生成
- 后续是否与其他来源报价单合并，另行讨论
- 平台费限免期起算节点与合同约定的保修起算节点相同
- 平台费限免期在合同中允许原样填写，但系统统一折算为月
- 只有收费方式为预付费或后付费时，才生成平台来源报价单
- 收费方式为免费时，不生成平台来源报价单
- 免费平台仍需保留免费原因或免费依据，便于后续追溯
