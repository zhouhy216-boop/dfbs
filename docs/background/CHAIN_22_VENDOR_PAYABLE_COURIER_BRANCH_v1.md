##  CHAIN_22_VENDOR_PAYABLE_COURIER_BRANCH_v1.md

## 1. 链条定位
快递商应付不是物流商应付主链的一部分，而是运输成本在快递供应商侧的一条独立应付与付款支链。

物流商应付解决的是：
大件运输 / 委托运输 / 物流合同计价 / 对账单付款

快递商应付解决的是：
系统内月付快递行为留痕 → 仓库录入快递单号与快递商 → 管理员导入单号费用 → 形成快递商月账单 → 付款

说明：
- 快递商应付与物流商应付分开管理
- 快递商应付当前只讨论**月付快递**
- 垫付快递不在本链条中展开，后续应补入报销支链
- 因此，本链条只承接快递供应商月结应付，不承接内部垫付报销

---

## 2. 本链条的业务目的
建立快递商应付支链的主要目的，不是单纯保存快递单号，而是把系统内发生的月付快递行为与快递费用建立稳定连接，形成供应商应付闭环。

当前主要目标包括：

1. 记录系统内发生的月付快递行为
2. 由仓库录入快递单号与快递商
3. 由管理员导入每个快递单号对应的费用
4. 用快递单号把费用与具体业务行为挂接
5. 形成快递商月账单
6. 完成发票 / 附件留档
7. 按账单执行付款
8. 跟踪哪些账单尚未付款

说明：
- 当前阶段首先强调“单号连接行为与费用”
- 当前阶段不把快递商应付定义成复杂计价系统
- 当前阶段也不把快递商应付扩展为垫付报销链

---

## 3. 快递商应付与成本支线的关系
快递商应付与运输成本有关联，但两者不是同一个对象。

当前冻结口径为：

1. 成本支线负责定义快递费 / 运费属于什么成本
2. 快递商应付负责处理这部分成本在快递供应商侧的记录、账单与付款
3. 快递行为、快递单号、快递费用、月账单、付款记录，共同构成快递商应付链条
4. 快递商应付不替代成本分类
5. 成本支线与快递商应付支链是一前一后的关系

说明：
- 成本支线回答“这是什么成本”
- 快递商应付支链回答“这笔成本如何挂接、如何形成账单、如何付款”

---

## 4. 快递商应付与垫付快递的关系
系统内快递行为当前分为两类：

1. 月付快递
2. 垫付快递

当前冻结口径为：

1. 月付快递进入快递商应付支链
2. 垫付快递不进入快递商应付支链
3. 垫付快递后续补入报销支链
4. 当前阶段不把月付快递与垫付快递混成同一付款逻辑
5. 当前版本只冻结月付快递的供应商应付口径

说明：
- 月付快递属于对快递商付款
- 垫付快递属于内部人员先垫资、后报销
- 两者对象和结算方向不同，因此不能混写

---

## 5. 纳入范围口径
当前阶段，快递商应付只纳入系统内发生的月付快递行为。

当前已知典型场景包括：

1. 零部件发给客户的月付快递费 / 运费
2. 零部件发给分仓的月付快递费 / 运费
3. 其他在系统内发生、且按月付方式结算的快递行为

说明：
- 当前阶段先冻结零部件发给客户、零部件发给分仓这两类典型场景
- 部分返回行为如果属于月付快递，也可纳入本链条
- 垫付返回不在本链条展开

---

## 6. 快递行为来源对象口径
快递商应付中的费用，不应脱离具体业务动作单独存在。

当前冻结口径为：

1. 每条快递费用都应尽量挂到某个具体快递行为
2. 快递行为首先来源于系统内的实际业务动作
3. 当前阶段，快递行为来源主要理解为：
   - 零部件发给客户
   - 零部件发给分仓
4. 快递费用不是脱离业务动作单独管理的抽象金额
5. 快递行为是快递单号与费用挂接的上游来源

说明：
- 快递商应付不只是账单管理
- 它还要回答“这笔快递费是因什么业务动作产生的”

---

## 7. 快递单号口径
快递单号是快递商应付中最核心的连接对象。

当前冻结口径为：

1. 仓库需要录入快递单号
2. 仓库需要同时录入对应快递商
3. 快递单号用于连接快递行为与快递费用
4. 一个快递单号只对应一次具体快递行为
5. 当前阶段不允许一张快递单号同时对应多笔业务行为

说明：
- 快递单号是行为与费用之间的桥
- 因此快递单号必须具备唯一挂接含义
- 若一张单号同时挂多笔行为，会破坏费用归属清晰度

---

## 8. 费用来源口径
快递商应付中的费用，当前通过管理员导入获得。

当前冻结口径为：

1. 管理员按快递商导入每个单号对应的费用
2. 当前阶段，快递费用不做系统自动推荐价
3. 当前阶段采用“单号金额导入”的方式
4. 快递费用首先以供应商账单导入值为基础
5. 当前版本不展开快递合同计价规则

说明：
- 快递商当前不按物流商那样做复杂合同推荐值计算
- 第一版先按账单金额导入理解最稳

---

## 9. 导入周期口径
快递商应付当前按月处理导入与账单。

当前冻结口径为：

1. 快递费用导入按某快递商、某自然月进行
2. 当前阶段不把多个快递商混在同一份月明细中处理
3. 月导入明细是快递商应付的原始来源
4. 快递商正式账单按自然月形成
5. 当前阶段快递商应付首先按月结型业务理解

说明：
- 快递商应付不是一票一付
- 第一版先按“月度导入 → 月账单”理解最稳

---

## 10. 挂接口径
快递商应付中的费用，需要通过快递单号挂接到具体行为。

当前冻结口径为：

1. 仓库录入的快递单号与快递商，构成快递行为留痕
2. 管理员导入的单号费用，通过快递单号与具体行为挂接
3. 系统需要能够看到每笔费用对应的是哪个业务行为
4. 系统也需要能够从具体行为反查对应快递费用
5. 当前阶段，快递单号是唯一挂接桥梁

说明：
- 快递商应付的核心不是“金额总计”
- 而是“金额能不能挂到正确行为上”

---

## 11. 未挂接费用口径
现实中，导入的快递费用可能暂时找不到系统内对应行为。

当前冻结口径为：

1. 若导入的快递单号在系统内找不到对应快递行为，可保留为未挂接费用
2. 当前阶段未挂接费用不强制丢弃
3. 管理员可后续将未挂接费用挂载到具体行为
4. 当前阶段先冻结“允许存在未挂接费用”这一原则
5. 当前版本不展开更复杂的异常处理流程

说明：
- 现实中可能存在仓库漏录、单号录错、导入先到等情况
- 因此第一版需要允许未挂接费用暂存

---

## 12. 月账单口径
快递商应付的正式账单对象，为快递商月账单。

当前冻结口径为：

1. 一个快递商 + 一个自然月 = 一张月账单
2. 月账单来源于该快递商该月份的单号费用明细
3. 月账单是正式付款对象
4. 当前阶段不把单号费用逐条作为付款对象
5. 月账单按月结型应付理解

说明：
- 单号是挂接桥梁
- 账单才是付款对象
- 这样更符合月结快递供应商的处理方式

---

## 13. 附件留档口径
快递商应付中的原始费用来源需要留档。

当前冻结口径为：

1. 导入的快递费用明细需要保留
2. 月账单需要能够关联原始导入明细作为留档依据
3. 当前阶段先冻结“存在留档”这一原则
4. 发票 / 附件需要与月账单关联留档
5. 当前版本不展开更细的附件管理方式

说明：
- 原始费用导入是追溯依据
- 发票与附件留档用于支持后续付款与核对

---

## 14. 付款口径
快递商应付当前按月账单付款，而不是按单号逐条付款。

当前冻结口径为：

1. 付款最小单位是月账单
2. 一次付款可以支付多个月账单
3. 当前阶段不允许对同一张月账单做部分付款
4. 账单状态当前按“未付款 / 已付款”理解
5. 系统需要能够看到哪些快递商账单尚未付款

说明：
- 付款对象是快递商月账单，不是单号明细
- 当前付款口径沿用物流商、平台商、SIM 卡商的统一方式

---

## 15. 与仓库动作的关系
快递商应付需要与仓库动作联动，但不是仓库主链本体。

当前冻结口径为：

1. 仓库是快递单号与快递商的录入来源
2. 仓库动作提供快递行为的业务基础
3. 快递商应付不替代仓库发货 / 调拨动作本身
4. 仓库动作解决“货怎么发出去”
5. 快递商应付解决“这次快递对应多少钱、怎么付款”

说明：
- 仓库动作与快递商应付是一前一后的关系
- 仓库动作提供业务来源
- 快递商应付提供费用与付款闭环

---

## 16. 当前边界
当前阶段，本链条只冻结月付快递的第一版业务口径，不展开以下内容：

1. 垫付快递报销规则
2. 更复杂的快递价格规则
3. 到付 / 现付 / 月结等更多结算模式细分
4. 单号重复、作废、异常扫描等复杂异常处理
5. 页面、字段、按钮、报表实现方式
6. 付款审批、银行执行、红字处理等细节

说明：
- 当前先把快递商月付应付的业务边界定义清楚
- 垫付快递后续应补入报销支链
- 当前版本不把两类快递结算方式混成一篇

---

## 17. 当前冻结结论
当前可冻结的总口径为：

- 快递商应付属于运输成本在快递供应商侧的应付与付款支链
- 快递商应付与物流商应付分开管理
- 当前系统内快递行为分为月付快递与垫付快递
- 本链条当前只冻结月付快递
- 垫付快递后续补入报销支链
- 当前纳入范围包括零部件发给客户、零部件发给分仓等系统内月付快递行为
- 部分返回行为若属于月付快递，也可纳入本链条
- 仓库需要录入快递单号与快递商
- 快递单号是快递行为与费用之间的核心连接对象
- 一个快递单号只对应一次具体快递行为
- 管理员按快递商、按月导入每个单号对应的费用
- 当前阶段快递费用不做自动推荐价
- 快递费用通过快递单号挂接到具体行为
- 找不到来源行为的费用可保留为未挂接费用
- 管理员可后续将未挂接费用挂载到具体行为
- 一个快递商 + 一个自然月 = 一张月账单
- 月账单是正式付款对象
- 付款最小单位是月账单
- 一次付款可以支付多个月账单
- 当前阶段不允许部分付款
- 账单状态当前按未付款 / 已付款理解
- 系统需要能看到哪些快递商账单尚未付款
- 原始导入明细、发票与附件需要留档
