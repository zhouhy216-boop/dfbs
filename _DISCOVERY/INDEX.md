# _DISCOVERY / INDEX
> 目的：集中存放“讨论到一半/暂停中的需求（THREAD）”，避免遗忘与重复沟通。  
> 使用方式：每个需求一份 THREAD 文件；以后继续讨论时，直接把对应 THREAD 文件上传/更新即可。

---

## 1) THREAD 索引（进行中 / 暂停）
| THREAD 文件 | 标题 | 状态 | 一句话说明 | 下次继续需要的实物 |
|---|---|---|---|---|
| `THREADS/THREAD_智能粘贴解析.md` | 智能粘贴解析 | 暂停 | 先做发货/委托表单，再做字段映射与解析扩展 | 发货/委托/工单字段清单 + 粘贴样本 + 委托类型清单 |

---

## 2) 新增 THREAD 的规范（统一、好找、不会乱）
- 存放路径：`_DISCOVERY/THREADS/`
- 文件命名：`THREAD_<需求名>.md`（例：`THREAD_发货表单.md`）
- 每个 THREAD 至少包含 4 段（建议照此写，方便续聊）：
  1) **已冻结**（不再反复讨论）
  2) **未确认（Unknown）**（下次继续的重点）
  3) **已给出的样本/例子**（原话或原文粘贴）
  4) **下次需要的实物**（字段清单/样本/截图/现有流程等）

---

## 3) 续聊规则（避免重复问）
- 继续讨论某个需求时：以对应 THREAD 文件为准，只推进“未确认/待实物”的部分。
- THREAD 里标为“已冻结”的内容：默认不再重复确认，除非你明确说要改。
