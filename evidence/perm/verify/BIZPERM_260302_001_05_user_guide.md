# BIZPERM Step-001-05 — 业务模块目录导入/导出（往返安全）证据与用户验证指南

**Parent:** BIZPERM-260302-001-05  
**Audience:** CEO / 初学者  
**Purpose:** 说明 Step-05 交付内容：**allowlist 超管**可通过 XLSX 对业务模块目录做**导出**与**安全导入**（预检 + 应用）；导出为双工作表完整结构，支持**往返不丢字段**；导入采用**预检（不写库）+ 应用（事务）**，**中文校验错误**可操作、**不静默破坏**；应用前需二次确认。本文档支持通过 UI 完成验证。

---

## 一、前置条件

- **入口**：**系统 → 账号与权限** → 账号 Tab → 点击某账号打开详情抽屉 → **业务模块视图** → **目录维护** Tab。仅 **allowlist 超管**可见「目录维护」及本步的「导出目录」「导入目录」按钮；普通 admin 无此 Tab，亦无法调用导出/导入接口（403）。
- **导出**：点击「**导出目录**」触发 GET `/api/v1/admin/bizperm/catalog/export`，浏览器下载 xlsx，文件名为中文（如 业务模块目录_yyyyMMdd_HHmm.xlsx）。
- **导入**：点击「**导入目录**」打开弹窗，先选 xlsx 文件，再点「**预检**」（仅校验，不写库），通过后可点「**应用导入**」（需确认后写库）。

---

## 二、UI 验证清单（≤12 条）

1. **导出下载与表头（A）**：以 allowlist 超管进入目录维护，点击「**导出目录**」；应下载一个 xlsx，文件名含中文「业务模块目录」及时间。打开文件：应有两个工作表「**节点**」「**操作点**」；节点表头为「节点ID、父节点ID、中文名称、排序」；操作点表头为「操作点ID、节点ID(空=未归类)、权限键、中文名称、排序、支持仅已处理(是/否)」。未归类行的节点ID 为空；支持仅已处理列为「是」或「否」。
2. **导入预检成功路径（B）**：在导入弹窗中选择**本系统刚导出的 xlsx**，点击「**预检**」；应显示「**预检通过，可应用导入**」及节点行数、操作点行数；此时「**应用导入**」按钮可点。
3. **导入预检失败与错误表（C）**：上传一份**含错误**的 xlsx（如重复节点ID、权限键不在权限树中、中文名称无中文字符等），点击「预检」；应显示「**预检未通过，请按下方错误修正后重新上传**」，并展示**错误表格**，列为「工作表、行、列、错误信息」，全部为中文；「应用导入」按钮**禁用**。预检**不写入数据库**，仅做校验。
4. **应用前需确认（D）**：预检通过后点击「**应用导入**」；应先弹出**确认框**：「应用导入将更新目录结构与操作点，确认继续？」；取消则关闭确认、不执行；确定后才发起 POST 应用并写库。
5. **应用成功与刷新（E）**：确认后若应用成功，应提示「**导入成功**」并展示本次节点更新数、操作点新建数、更新数；弹窗关闭，**目录维护界面自动刷新**（重新拉取目录树与操作点），所见即所得。
6. **应用失败展示错误（F）**：若应用时后端校验仍不通过（如预检通过后有人改库导致冲突），返回 400；前端应提示「**导入失败，请按错误修正文件**」，并在弹窗内展示**同一套错误表格**（工作表/行/列/错误信息），**不静默破坏**、无部分写入。
7. **往返不丢字段（G）**：导出 → 用 Excel 仅做少量可接受修改（如改某节点中文名称）→ 保存 → 再导入（预检通过后应用）；再次导出，对比两次 xlsx：**结构、映射、中文名称、排序、支持仅已处理** 应与预期一致，不丢字段。未在导入文件中的节点/操作点**不会被删除**（v1 不做“按文件全量替换”）。
8. **非 allowlist 不可见（H）**：以**非 allowlist** 的 admin 登录，进入账号与权限 → 某账号 → 业务模块视图；应**只有「按账号分配」**，无「目录维护」Tab，故无「导出目录」「导入目录」按钮。若直接调 GET/POST 导出/导入接口，应返回 **403**，前端可提示「无权限（仅超管可导入/导出）」。
9. **全中文**：导出文件名、表头、导入弹窗内所有文案、预检结果、错误表列名与错误信息、确认框、成功/失败提示均为中文；无英文占位。
10. **仅 xlsx**：导入仅接受 `.xlsx` 文件，选择文件后需点击「预检」再决定是否「应用导入」；预检与应用使用同一文件，避免中途换文件未预检即应用。
11. **节点 v1 仅更新**：导入时节点为**按节点ID 更新**（中文名称、父节点ID、排序），**不创建新节点、不删除缺失节点**；操作点为**按权限键 upsert**（新建或更新）。未知权限键会被预检/应用拒绝并报中文错误。
12. **审计（可选）**：成功导出后，审计中可有 `BIZPERM_CATALOG_EXPORT`；成功应用导入后，审计中可有 `BIZPERM_CATALOG_IMPORT`，备注为中文（如「导入业务模块目录：节点N、操作点新建X、更新Y」）。可查 GET `/api/v1/admin/perm/audit?limit=20` 等做抽查。

## 三、说明与约束

- **预检 + 应用**：预检（POST `/import/preview`）**只读**，不写库；应用（POST `/import/apply`）在**同一事务**内先更新节点再 upsert 操作点，校验失败则 400、**不部分提交**，保证**不静默破坏**。
- **中文校验**：所有校验错误以**中文**返回（工作表、行、列、错误信息），便于按表修正；未知权限键、重复节点ID/权限键、父节点环、中文名称须含中文等均有对应中文提示。
- **往返不丢字段**：导出包含节点 ID/父节点ID/中文名称/排序，操作点 ID/节点ID/权限键/中文名称/排序/支持仅已处理；导入后再次导出，上述字段应完整、一致（除 ID 由库生成或保留外，业务字段可往返）。
- **v1 安全策略**：节点为**仅更新已有节点**；操作点按 **permission_key** 做 upsert；**不删除**“文件中未出现”的节点或操作点，避免误删。

---

## 四、CEO 一句话验证要点

1. **导出完整、导入安全**：超管在目录维护中可「导出目录」得到双工作表 xlsx（节点+操作点，中文表头）；「导入目录」为先「预检」后「应用」、应用前有确认；错误全部以中文表格展示，可按表修正，无静默破坏。
2. **往返不丢字段**：导出 → 适度修改 → 导入（预检通过并应用）→ 再导出，结构/映射/标签/排序/支持仅已处理等与预期一致；未在文件中的项不会被删除。
3. **门控与审计**：仅 allowlist 超管可见并可使用导出/导入；非超管 403；可选核对审计中的导出/导入记录。
