# BIZPERM Step-001-04 — 数据范围（全量 / 仅已处理）证据与用户验证指南

**Parent:** BIZPERM-260302-001-04  
**Audience:** CEO / 初学者  
**Purpose:** 说明 Step-04 交付内容：在「**按账号分配**」视图中，支持每操作点两种数据范围——**全量**与**仅已处理**；范围开关仅对目录中标记为支持的操作点（op.handledOnly=true）显示且可切换，不支持的操作点明确标注「仅全量（不支持仅已处理）」；保存/还原同时持久化权限勾选与范围设置，重开抽屉后可恢复。本文档支持通过 UI 完成验证。

---

## 一、前置条件

- **入口**：**系统 → 账号与权限** → 账号 Tab → 点击某账号打开详情抽屉 → **业务模块视图** 区块 → **按账号分配**（默认 Tab 或唯一内容）。admin 与 allowlist 超管均可使用按账号分配；超管另可见「目录维护」Tab，本步不涉及目录维护，仅涉及按账号分配内的范围开关。
- **两种范围**：**全量**（ALL）：该操作权限下可见/操作全部数据；**仅已处理**（HANDLED_ONLY）：该操作权限下仅针对用户**参与过流程/历史的记录**（“已处理/参与过”的通俗理解）；**不承诺**所有业务模块均已实现“仅已处理”的过滤逻辑，本步仅做**开关与存储**，具体计算可逐模块后续完善。
- **支持范围切换的操作点**：仅当目录中该操作点标记为**支持仅已处理**（op.handledOnly=true）时，在**已勾选**该操作点的前提下，才会显示「全量 / 仅已处理」分段选择；未支持的操作点仅显示灰色文案「仅全量（不支持仅已处理）」。

---

## 二、UI 验证清单（≤12 条）

1. **支持仅已处理的操作点（A）**：在按账号分配视图中，对目录内 **handledOnly=true** 的操作点，**先勾选**该操作点；勾选后该行应出现**范围选择**（分段控件）：选项为「**全量**」「**仅已处理**」，默认或已保存为全量时显示「全量」，可切换为「仅已处理」；**未勾选**时该行不显示范围选择（仅显示复选框与名称及「仅已处理」标签）。
2. **不支持的操作点（B）**：对目录内 **handledOnly=false** 的操作点，该行**不显示**范围选择，仅显示灰色小字「**仅全量（不支持仅已处理）**」，表示该操作点仅支持全量数据范围，不可切换。
3. **保存持久化权限与范围（C）**：点击「**保存**」后，除原有 PUT override（roleTemplateId=null、addKeys=勾选、removeKeys=[]）外，会向 **PUT `/api/v1/admin/bizperm/scope/accounts/{userId}/scopes`** 提交当前**已勾选且范围为「仅已处理」**的操作点，以及需要从「仅已处理」改回「全量」的项；保存成功后，再次打开该账号详情或刷新，勾选状态与范围选择应与保存一致（从服务端 override 与 scopes 接口恢复）。
4. **还原同时恢复勾选与范围（D）**：点击「**还原**」后，草稿的**勾选**与**范围选择**均恢复为**上次从服务端加载**的状态（即打开抽屉或上次保存后的快照）；未保存的权限或范围修改均丢弃。
5. **筛选「只看仅已处理范围」（E）**：勾选筛选「**只看仅已处理范围**」后，列表仅显示**当前已勾选且范围设为「仅已处理」**的操作点；取消该筛选后恢复为原有列表逻辑。
6. **汇总行（E）**：按账号分配视图中有一行汇总文案：「**仅已处理范围：X 项（仅对支持的操作点生效）**」，X 为当前**已勾选、且支持仅已处理、且范围选为「仅已处理」**的操作点数量；修改勾选或范围后 X 实时更新，保存后重开抽屉，X 与持久化结果一致。
7. **未归类操作点同样适用**：未归类/待认领区块中的操作点，若 handledOnly=true 且已勾选，同样显示「全量/仅已处理」分段；若 handledOnly=false，同样显示「仅全量（不支持仅已处理）」。
8. **模板→直接分配确认（F，可选）**：若账号当前为「角色模板+覆盖」，在按账号分配中点击保存仍会先弹出确认「切换为按账号直接分配并清除模板依赖」；确认后同时保存权限勾选与范围设置，行为与 Step-03 一致，不受本步范围功能影响。
9. **dirty 与保存/还原按钮**：只要**勾选**或**范围**与上次加载不一致，即视为有未保存修改，「保存」可点、「还原」可点；保存或还原后，两者恢复一致，保存变灰、还原变灰（无未保存修改时）。
10. **全中文**：范围相关文案均为中文（「全量」「仅已处理」「仅全量（不支持仅已处理）」「只看仅已处理范围」「仅已处理范围：X 项（仅对支持的操作点生效）」等）；无英文占位或标签。
11. **门控**：范围接口与按账号分配入口一致，需 admin 或 super-admin；非管理员无法进入账号与权限页，亦无法调用 scope 接口。
12. **不承诺占位/未实现模块**：目录中若存在占位或尚未实现“仅已处理”过滤的业务模块，本步**不承诺**其数据过滤效果；仅保证开关可见性、存储与恢复正确；具体业务含义由各模块后续实现。

## 三、后端参考

- **范围读写**：**GET** `/api/v1/admin/bizperm/scope/accounts/{userId}/scopes` 返回该账号当前每 permissionKey 的范围（ALL / HANDLED_ONLY）；**PUT** 同路径，请求体 `{ updates: [ { permissionKey, scope } ] }`，用于保存范围。门控：AdminOrSuperAdminGuard；仅当目录中 op.handledOnly=true 时允许 scope=HANDLED_ONLY，否则 400 BIZPERM_SCOPE_NOT_SUPPORTED。
- **权限覆盖**：仍为 GET/PUT `/api/v1/admin/account-permissions/accounts/{userId}/override`（与 Step-03 一致）；保存时先写 override，再按需写 scopes。

## 四、说明与约束

- **仅已处理含义**：本步将「仅已处理」解释为**用户参与过该记录的流程/历史**（“已处理/参与过”），便于理解；**不承诺**所有模块均已支持该语义，占位或未实现模块可能仅做开关与存储，实际过滤逻辑可逐模块后续完善。
- **不承诺占位模块**：若目录中存在占位或未对接业务逻辑的操作点，仅保证「仅全量（不支持仅已处理）」或范围开关的展示与存储正确，不承诺前端/后端对该 key 的“仅已处理”数据过滤已实现。
- **本步范围**：仅做**范围开关的展示、保存、还原与接口存储**；不改变权限校验（Step-05 等）逻辑，不改变目录维护（allowlist 超管）逻辑。

---

## 五、CEO 一句话验证要点

1. **范围仅对支持的操作点可见**：在账号详情 → 业务模块视图 → 按账号分配中，**支持仅已处理**的操作点勾选后出现「全量/仅已处理」分段；**不支持**的操作点明确显示「仅全量（不支持仅已处理）」。
2. **保存与还原含范围**：保存后权限勾选与范围一并持久化，重开抽屉可恢复；还原则同时恢复勾选与范围到上次加载状态；筛选「只看仅已处理范围」与汇总「仅已处理范围：X 项」可辅助核对。
3. **不承诺占位/未实现模块**：全量/仅已处理的**开关与存储**本步完成；「仅已处理」的业务含义（如按“已处理/参与过”过滤）可逐模块后续完善，本步不做承诺。
