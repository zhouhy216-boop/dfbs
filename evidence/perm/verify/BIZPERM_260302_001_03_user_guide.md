# BIZPERM Step-001-03 — 按账号直接分配（业务模块视图）证据与用户验证指南

**Parent:** BIZPERM-260302-001-03  
**Audience:** CEO / 初学者  
**Purpose:** 说明 Step-03 交付内容：在账号编辑抽屉「**业务模块视图**」中，**admin** 可使用「**按账号分配**」视图，以业务模块目录为清单勾选操作点（按钮/动作级），保存为**直接分配**（不依赖角色模板：roleTemplateId=null，addKeys=勾选，removeKeys=[]）；支持搜索与筛选、保存与还原；若账号当前为「角色模板+覆盖」，保存前会弹出确认以切换为直接分配。**allowlist 超管**另可见「目录维护」Tab；**非超管 admin** 仅见「按账号分配」。差异视图（高级）默认折叠，不影响回归。本文档支持通过 UI 完成验证。

---

## 一、前置条件

- **入口**：系统菜单 **「系统」→「账号与权限」**，路由 `/admin/account-permissions`；仅 **admin 或 super-admin** 可进入。
- **admin 与 allowlist 超管区别**：**admin（非 allowlist）** 在业务模块视图中**仅看到「按账号分配」**内容（无 Tab 切换），目录数据来自 GET `/api/v1/admin/bizperm/catalog/read`（只读），可勾选/取消勾选操作点并保存、还原。**allowlist 超管**在业务模块视图中看到 **Tab：「按账号分配」|「目录维护」**，默认「按账号分配」；可切换到「目录维护」进行目录树与操作点维护（与 Step-02 一致）。

---

## 二、UI 验证清单（≤12 条）

1. **Admin 仅见「按账号分配」（A）**：以**非 allowlist** 的 admin 登录，进入「账号与权限」→ 账号 Tab → 点击某账号打开详情抽屉；「业务模块视图」区块内**仅**显示按账号分配内容（搜索框、筛选、目录节点与操作点列表、未归类区块、保存/还原），无「目录维护」Tab，无目录树编辑与认领按钮。
2. **搜索与筛选（B）**：在按账号分配视图中，**搜索框**可输入关键字，过滤「模块名、操作名或权限键」；**筛选**包含三个复选框：「只看已勾选」「只看未归类」「只看仅已处理」。勾选后列表仅显示符合条件项；列表为目录树结构（节点下操作点）+ 未归类/待认领区块，每行有复选框与中文名（缺省显示 permissionKey）及「仅已处理」标签。
3. **勾选/取消勾选**：每个操作点行有**复选框**，勾选即加入当前草稿的“已勾选”集合，取消即移除；保存前为草稿，还原可恢复为上次加载状态。
4. **保存写入直接分配并持久化（C）**：点击「**保存**」后，请求 PUT `/api/v1/admin/account-permissions/accounts/{userId}/override`，请求体为 `roleTemplateId: null`、`addKeys: [当前勾选的所有 permissionKey]`、`removeKeys: []`。保存成功后，再次打开该账号详情或刷新，按账号分配视图中的勾选状态应与保存一致（即从服务端重新拉取的 addKeys 与上次保存一致）；**不依赖角色模板**，仅凭 addKeys 即可还原界面。
5. **还原恢复上次加载（D）**：点击「**还原**」后，草稿勾选恢复为**上次从服务端加载**的 addKeys 状态（即打开抽屉或保存后刷新时的快照），未保存的修改丢弃。
6. **当前为「角色模板+覆盖」时保存需确认（E）**：若该账号当前 override 的 `roleTemplateId` 非空（即使用角色模板+追加/移除覆盖），按账号分配视图顶部会显示**警告条**：「当前账号使用角色模板 + 覆盖；本视图保存将切换为「按账号直接分配」并清除模板依赖。」点击「保存」时先弹出**确认弹窗**，文案为「本视图保存将切换为「按账号直接分配」并清除模板依赖。确定保存？」；用户确认后执行保存（roleTemplateId=null, addKeys=勾选, removeKeys=[]），保存成功后该账号即切换为直接分配，不再依赖模板。
7. **超管可切换「按账号分配」与「目录维护」（F）**：以 allowlist 超管登录，业务模块视图中出现两个 Tab：「按账号分配」（默认）、「目录维护」。切换至「目录维护」时行为与 Step-02 一致（目录树、操作点编辑、未归类认领等）；切换回「按账号分配」时仍为勾选清单与保存/还原，两者互不覆盖逻辑。
8. **「差异视图（高级）」默认折叠（F）**：账号详情抽屉内，在业务模块视图区块**下方**的「**差异视图（高级）**」为 Collapse，**默认折叠**（不展开）。展开后仍为原有差异视图（保存/还原、分配角色模板、继承/追加/移除、最终生效、高级编辑）；默认折叠确保按账号分配为主流程，且不破坏原有差异视图与高级编辑功能。
9. **无英文标签**：按账号分配视图内所有标签、占位符、按钮、空态、确认文案均为中文（如「搜索模块名、操作名或权限键」「只看已勾选」「保存」「还原」「暂无目录数据」「确认保存」等）。
10. **错误与门控**：非 admin 无法进入「账号与权限」页；调用 catalog/read 或 override 若返回 403，前端提示「无权限」或后端返回的中文 message。
11. **角色模板可选**：切换为直接分配后，该账号的权限仅由 addKeys 决定；角色模板不再参与计算。若之后在「差异视图（高级）」中再次选择角色模板并保存，则又回到「模板+覆盖」模式；在按账号分配中再次保存仍会提示确认后写回直接分配。即**角色模板与直接分配二选一**，不阻塞、不混用同一套存储。
12. **直接分配 v1 无“移除”概念**：按账号分配视图中仅存在「勾选=有权限、不勾选=无权限」；保存时 `removeKeys` 恒为 `[]`，**移除优先**概念不适用于直接分配 v1。

---

## 三、说明与约束

- **存储规则**：按账号分配保存时固定为 `roleTemplateId=null`、`addKeys=当前勾选的 permissionKey 列表`、`removeKeys=[]`；加载时若 `roleTemplateId==null` 则用 `addKeys` 作为勾选集合，若 `roleTemplateId!=null` 则显示警告条并在保存时要求确认后切换。
- **角色模板为可选**：使用直接分配后不依赖任何角色模板；如需再启用模板，可通过下方「差异视图（高级）」操作。直接分配 v1 不做“移除”语义，仅 addKeys 有效。
- **数据源**：目录来自 GET `/api/v1/admin/bizperm/catalog/read`（AdminOrSuperAdminGuard）；覆盖读写为 GET/PUT `/api/v1/admin/account-permissions/accounts/{userId}/override`（同上门控）。无需新增接口。

---

## 四、CEO 一句话验证要点

1. **按账号分配为主、门控清晰**：Admin 在账号详情抽屉的「业务模块视图」中**仅**使用「按账号分配」：按目录勾选操作点，搜索/筛选后保存即写入直接分配（无角色模板）；还原恢复上次加载。超管可额外切换「目录维护」Tab。
2. **模板+覆盖时保存需确认**：若账号当前为角色模板+覆盖，在按账号分配中点击保存会先弹出确认「切换为按账号直接分配并清除模板依赖」，确认后持久化；再次打开该账号可见勾选与保存一致。
3. **差异视图（高级）默认折叠、回归不受影响**：下方「差异视图（高级）」默认折叠，展开后仍为原保存/还原、角色模板、差异视图与高级编辑；直接分配与差异视图两套逻辑并存，互不破坏。
