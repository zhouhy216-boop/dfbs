# BIZPERM Step-001-02 — 业务模块目录维护与未归类自动发现 证据与用户验证指南

**Parent:** BIZPERM-260302-001-02  
**Audience:** CEO / 初学者  
**Purpose:** 说明 Step-02 交付内容：在账号编辑抽屉「**业务模块视图**」中，**allowlist 超管**可使用目录维护 UI（中文树、节点增删改移/排序、操作点中文名/排序/仅已处理、未归类认领）；**未归类**自动包含「权限全集 − 已持久化操作点」的缺失 key（仅展示、不做语义自动归类）；认领后即可编辑。入口为 admin-only，维护 UI 仅超管可见；界面与提示**全中文**。本文档支持通过 UI 完成验证。

---

## 一、前置条件

- **入口**：系统菜单 **「系统」→「账号与权限」**，路由 `/admin/account-permissions`；仅 **admin 或 super-admin** 可进入（由 AdminOrSuperAdminGuard 门控）。
- **维护 UI 可见性**：**allowlist 超管**（`/api/v1/admin/perm/super-admin/me` 返回 `allowed: true`）在账号详情抽屉的「业务模块视图」中可见**完整目录维护 UI**（目录树 + 操作点面板 + 未归类/待认领）；**非 allowlist** 管理员仅见占位骨架与「目录维护仅超管可用」说明，无树与未归类列表。
- **如何制造新权限键以验证自动发现**：在「账号与权限」→「**权限管理**」Tab（仅 allowlist 可见）中：**新建模块**（填写模块 key、显示名称、父节点等）或对已有模块**设置动作**（勾选新的 action），保存后即扩展「权限全集」。随后在「账号」Tab 打开任意账号详情抽屉，在业务模块视图中刷新或重新打开，**未归类/待认领**列表应出现新出现的 `moduleKey:actionKey`（来自计算、id 为空），无需手动录入；**不做语义自动归类**，仅自动进入未归类供超管认领。

---

## 二、UI 验证清单（≤12 条）

1. **维护 UI 仅超管可见（A）**：以 allowlist 超管登录，进入「账号与权限」→ 账号 Tab → 点击某账号打开详情抽屉；「业务模块视图」区块内显示**目录树**、**当前节点操作点**表格、**未归类/待认领**面板及「从未归类添加」「一键认领到当前节点」等操作。以非 allowlist 管理员登录同路径，同一区块仅显示占位卡片与「目录维护仅超管可用」，无树与未归类。
2. **节点操作（B）**：在目录树中可**新建根节点**、对任意节点**新建子节点**（中文名称必填）；可对节点**重命名**（弹窗改中文名）；可**删除**节点（有子节点或有关联操作点时不可删，前端灰显/提示或 Popconfirm 说明）；可**上移/下移**同层兄弟排序（根节点顺序若未开放则仅子节点可排）。所有操作立即请求后端并刷新目录。
3. **操作点操作（C）**：选中某节点后，右侧展示该节点下操作点列表；可**编辑中文名称**（点击名称进入编辑，失焦或回车保存）；可**展开/收起**权限键用于查看与复制；可切换**仅已处理**开关；可**上移/下移**调整同节点内排序。操作立即生效并刷新。
4. **未归类列表与搜索（D）**：未归类面板展示「未归类/待认领」列表（持久化 `node_id` 为空的项 + 计算得出的缺失 key）；支持**搜索**（按权限键或中文名过滤）；空态文案为「**暂无未归类权限**」；所有标签与提示为中文。
5. **认领与从未归类添加（D）**：在未归类列表中勾选若干项，点击「**一键认领到当前节点**」（需先选中左侧节点）；或选中节点后点击「**从未归类添加**」在弹窗中勾选未归类项并认领。认领后这些操作点从未归类消失并出现在该节点下，可进行编辑中文名、排序、仅已处理等操作。
6. **自动发现新 key 进未归类（E）**：在权限管理 Tab 中新增模块或为模块增加动作后，返回账号 Tab 打开业务模块视图（或刷新），未归类列表应**自动出现**新产生的 `moduleKey:actionKey`（接口返回项无持久化 id，即 id=null）；**不做语义自动归类**，不会自动挂到任何节点。勾选该 key 并「认领到当前节点」后，后端会创建对应操作点并挂到节点，此后可正常编辑、排序、设仅已处理。
7. **认领使计算项可编辑（E）**：对 id=null 的未归类项执行认领后，其会从未归类移除并出现在目标节点的操作点列表中，且带有持久化 id，可编辑中文名、排序、仅已处理。
8. **无英文标签**：目录树、操作点表头、未归类面板、按钮、空态、弹窗标题与确认文案等均为中文（如「目录树」「中文名称」「权限键」「仅已处理」「排序」「未归类/待认领」「从未归类添加」「一键认领到当前节点」「暂无未归类权限」）；错误提示为中文或后端返回的中文 message。
9. **错误与门控**：403 时提示「无权限（仅超管可维护）」；400 时展示后端返回的中文 message（如权限键不存在、节点有子节点或操作点不可删等）。
10. **入口与页面门控**：非 admin 用户不应看到「账号与权限」菜单；直接访问 `/admin/account-permissions` 应被重定向或拦截。
11. **与权限区无冲突**：业务模块视图维护区块与下方「保存/还原、分配角色模板、差异视图、高级编辑」等账号覆盖区独立；维护操作不改变账号覆盖的草稿与保存/还原行为。
12. **审计（可选）（F）**：在「权限管理」或系统审计入口（若提供）中查询近期操作，可看到 BIZPERM_* 类型（如 BIZPERM_CATALOG_NODE_CREATE、BIZPERM_OP_CLAIM 等），备注为中文（如「节点：xxx」「认领 n 个操作点至节点」）。

## 三、说明与约束

- **不做语义自动归类**：系统**不会**根据权限键含义自动将其挂到某个目录节点；新发现的 key 仅加入「未归类/待认领」列表，由超管手动认领到目标节点。
- **未归类 = 持久化未归类 + 计算缺失 key**：**持久化未归类**指表 `biz_perm_operation_point` 中 `node_id` 为空的记录；**计算缺失 key** 指当前权限全集（来自权限树模块×动作）中尚未在表中出现过的 `moduleKey:actionKey`。GET 目录接口将二者合并返回，计算项 id 为 null，仅用于展示与认领；认领时若 key 尚未持久化则后端会插入新行并挂到目标节点。
- **数据源与接口**：目录与未归类来自 `GET /api/v1/admin/bizperm/catalog`；维护写操作使用同路径下的 nodes、op-points、claim、handled-only 等接口；全部需 allowlist 超管身份。

---

## 四、CEO 一句话验证要点

1. **维护能力与门控**：在「系统 → 账号与权限」→ 账号 Tab 打开某账号详情抽屉后，仅 **allowlist 超管**能看到完整「业务模块视图」维护 UI（目录树 + 操作点 + 未归类）；可对节点增删改移/排序，对操作点改中文名、排序、仅已处理，以及从未归类认领到节点；非超管仅见占位与「目录维护仅超管可用」。
2. **自动发现与认领**：在权限管理中新增模块或动作后，新 key 会**自动**出现在未归类列表（id 为空），**不做语义自动归类**；超管可勾选后「一键认领到当前节点」，认领后可正常编辑该操作点。
3. **全中文与无回归**：界面标签、空态、错误提示均为中文；业务模块视图与下方账号覆盖（保存/还原、差异视图）互不影响。
